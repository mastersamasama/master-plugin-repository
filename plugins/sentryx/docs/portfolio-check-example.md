# SentryX 持仓风险检查 — 使用示例

```

```
check if any of my tokens are risky

## 风险判定标准

### 合约层面 — 高风险 (High Risk)

| 检查项 | 触发条件 |
|--------|----------|
| 蜜罐 (Honeypot) | 可买入但无法卖出 |
| 增发后门 (Mintable) | 合约拥有者可无限铸造代币 |
| 隐藏交易税 > 10% | 买/卖税率超过 10% |
| 未验证代理合约 | 可升级合约指向未公开的实现代码 |

### 合约层面 — 中风险 (Medium Risk)

| 检查项 | 触发条件 |
|--------|----------|
| 所有权未放弃 | Meme 币的合约仍可被所有者修改 |
| 代理合约未验证 | 可升级但实现代码未经验证 |

### 市场层面

| 检查项 | 触发条件 | 风险等级 |
|--------|----------|----------|
| 持仓集中度 | Top 10 持有者 > 80% 供应量 | Medium |
| 流动性不足 | DEX 总流动性 < $1,000 | Medium |
| 无交易量 | 24h 交易量接近 0（非新币） | Low |

## 可能的后续操作

根据扫描结果，用户可选择：

| 场景 | 建议操作 |
|------|----------|
| 发现高风险代币 | `卖出高危Token` 或 `sell risky tokens` |
| 发现高风险授权 | `撤销高危授权` 或 `revoke risky approvals` |
| 全部安全 | 无需操作，建议定期复查 |
| 需要更多细节 | `详细` 或 `full details` |
